Architecture Explanation

DataSense implements a goal-directed agentic loop using Claude claude-sonnet-4 as the planning and reasoning core. The agent uses Anthropic's tool-use API to take real actions — executing Python code and producing charts — rather than just generating text.

01 · Perception

Reads the CSV and builds a rich context: shape, column types, statistical summary, missing values, and the first 5 rows. This is passed verbatim to the LLM as its "view of the world."

02 · Planning (LLM)

Claude autonomously decides which analyses to perform and in what order, based on the data context and the user's stated goal. No hardcoded analysis steps — the plan emerges from reasoning.

03 · Tool Execution

Two tools: execute_python runs sandboxed pandas/matplotlib code and saves charts as PNGs; finish_report ends the loop and returns the final narrative report.

04 · Memory & Safety

A memory dict tracks completed steps, generated charts, errors, and retry counts. If code fails, the error is fed back to the LLM for self-correction. Max 12 iterations prevents runaway loops.

Agentic Loop Flow

CSV + goal→ build context→ Claude plans→ execute tool→ check result→ retry / continue→ finish report

Agent Concepts Demonstrated

Concept Implementation in DataSense
Perception CSV is parsed and converted to a structured context block (shape, dtypes, head, stats, null counts) sent to the LLM as its sensory input.
Decision-making Claude claude-sonnet-4 autonomously selects which analyses to run — no fixed analysis pipeline. Decisions are driven by the data and user goal.
Action The execute_python tool runs real Python code (pandas, matplotlib) in a sandboxed namespace. Results are real outputs — charts saved to disk, statistics printed to stdout.
Memory A Python dict tracks completed steps, generated chart paths, all errors encountered, and per-step retry counts across the entire loop.
Error recovery If code execution fails, the full error traceback is fed back into the conversation. Claude reads the error and rewrites the code. Max 2 retries per step.
Safety Code runs in a restricted namespace with a whitelisted __builtins__ dict. Dangerous functions (file I/O, network, imports) are not available. plt.savefig is patched to force output to the session directory.
Goal-directed loop The agent continues until finish_report is called or 12 iterations are reached — a classic sense-plan-act loop with a terminal condition.
